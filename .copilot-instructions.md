# GitHub Copilot Instructions for Portfolio Project

You are assisting with a personal portfolio website built with Astro 3, React, and Tailwind CSS. This site showcases professional work experience, side projects, photography, and personal interests, featuring interactive elements like an AI chatbot (powered by Cohere) and data visualizations for music preferences and coding statistics.

## Key Technologies & Architecture

- **Astro 3**: Main framework using Islands Architecture for optimal performance
- **React 18**: Used for interactive components via Astro's React integration
- **Tailwind CSS + DaisyUI**: Styling with DaisyUI component library (themes: `pastel`, `dim`)
- **TypeScript**: Type-safe development across `.astro`, `.tsx`, and `.ts` files

## Development Commands

- `bun dev` or `bun start` - Start development server at http://localhost:4321
- `bun run build` - Build for production (outputs to `dist/`)
- `bun run preview` - Preview production build locally
- `bun run pretty` - Format code with Prettier
- `bun run gen-resume-pdf` - Generate PDF from `src/pages/work/resume.md`

## Important Patterns & Conventions

### Layouts
- `Layout.astro`: Main wrapper with theme switching, navigation, footer, and chat modal
- Uses Astro's `ViewTransitions` for smooth page navigation
- Theme persistence via `localStorage` with DaisyUI themes (`pastel`, `dim`)

### Interactive Components (React)
- `ChatWithHistory.tsx`: AI chatbot with streaming responses, conversation history in `localStorage`
- `Carousel.tsx` / `FullCarousel.tsx`: Image carousels
- `WordCloud.tsx`: Spotify genre visualization using `react-d3-cloud`
- `Coding.tsx`: Wakatime coding stats using Chart.js
- Use `client:only="react"` or `client:visible` directives in Astro for hydration

### Data Fetching
- S3 bucket for static data (Spotify artists JSON)
- `CodingWrapped.astro`: Fetches Wakatime data from Guac API (requires env vars)
- Photography images in `src/data/travelImages.ts` and S3

### Styling
- Tailwind utility classes throughout
- DaisyUI components (badges, cards, buttons, modals, collapse, etc.)
- Theme switching uses `data-theme` attribute on `<html>`
- Custom CSS in `src/assets/styles/resume.css` for PDF generation

### Content Management
- Work tiles configured as array in `src/pages/work/index.astro`
- Photo galleries use typed arrays from `src/data/travelImages.ts`
- Resume in markdown with frontmatter layout

## Environment Variables
- `GUAC_URL`, `GUAC_USER`, `GUAC_PASS`: Wakatime API authentication
- Accessed via `import.meta.env`

## Best Practices
- Use Astro Islands pattern for performance
- Follow TypeScript types from `src/types/`
- Test builds with `bun run build` before committing
- Keep interactive components minimal and use appropriate client directives
- Update data files for new content rather than hardcoding in components

## Common Tasks
- Adding a new page: Create `.astro` file in `src/pages/`, wrap in `<Layout>`
- Adding React component: Create `.tsx` in `src/components/`, import with `client:*` directive
- Updating photography: Modify `src/data/travelImages.ts` and year pages
- Resume updates: Edit `src/pages/work/resume.md` and regenerate PDF</content>
